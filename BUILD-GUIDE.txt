================================================================================
                              SYSTEM OVERVIEW
================================================================================

The Power Hub is a custom CMS (Content Management System) built for small to
medium businesses. It provides an easy-to-use admin portal for managing website
content without technical knowledge.

Target Clients:
- Chambers of Commerce
- Small businesses
- Nonprofits
- Professional services firms
- Any organization needing a managed website with admin access

================================================================================
                              TECH STACK
================================================================================

FRONTEND:
- Framework: Next.js 14+ (App Router)
- Styling: Tailwind CSS
- Language: TypeScript
- Icons: Lucide React
- Animations: Framer Motion (optional)
- Image Handling: Next.js Image component

BACKEND:
- API Routes: Next.js Route Handlers
- Database: Supabase (PostgreSQL)
- File Storage: Supabase Storage + GitHub (for content JSON)
- Authentication: Custom (stored in Supabase or localStorage)

DEPLOYMENT:
- Hosting: Vercel
- Repository: GitHub
- CDN: Vercel Edge Network (automatic)

INTEGRATIONS (Optional):
- AI: Claude API (for AI Assist feature)
- Calendar: Google Calendar API + Embed
- Forms: Go High Level webhooks
- Analytics: Custom scripts via Scripts manager

Each feature is modular and can be included or excluded based on client needs.

------------------------------------------------------------------------------
CORE FEATURES (Always Include)
------------------------------------------------------------------------------

[ ] DASHBOARD
    - Overview of site content
    - Quick stats (content files, media count)
    - Recent activity
    - Quick action buttons
    Files: app/power-hub/dashboard/page.tsx

[ ] CONTENT EDITOR
    - Edit JSON content files for each page
    - Visual form-based editing (no code required)
    - Auto-save to GitHub
    - Live preview capability
    Files: app/power-hub/dashboard/content/
    Dependencies: GITHUB_TOKEN env variable

[ ] MEDIA LIBRARY
    - Upload images
    - Browse uploaded media
    - Copy URLs for use in content
    - Delete unused media
    Files: app/power-hub/dashboard/media/
    Dependencies: GITHUB_TOKEN env variable

[ ] LOGIN SYSTEM
    - Secure login page
    - Configurable username/password
    - Session management (localStorage)
    Files: app/power-hub/page.tsx

[ ] SETTINGS
    - Change login credentials
    - Site configuration
    Files: app/power-hub/dashboard/settings/
    Dependencies: Supabase (portal_settings table)

------------------------------------------------------------------------------
OPTIONAL FEATURES (Client Choice)
------------------------------------------------------------------------------

[ ] AI ASSIST
    - Claude-powered content improvement
    - Rewrite, expand, summarize content
    - Grammar and tone adjustments
    Files: app/power-hub/dashboard/ai/
    Dependencies: Client's Claude API key (entered in browser)
    Remove if: Client doesn't want AI features

[ ] MEMBERS DIRECTORY
    - Manage member/client listings
    - Tiers (Sponsor, Ambassador, Member)
    - Logo uploads
    - Active/hidden status
    Files: app/power-hub/dashboard/members/
    Dependencies: Supabase (members table, member-logos bucket)
    Remove if: Not a membership organization

[ ] SCRIPTS MANAGER
    - Add tracking scripts (GTM, Meta Pixel, etc.)
    - Header or footer placement
    - Sitewide or page-specific
    - Enable/disable without removing
    Files: app/power-hub/dashboard/scripts/
    Dependencies: Supabase (custom_scripts table)
    Remove if: Client handles their own tracking

[ ] CALENDAR
    - Embedded Google Calendar
    - View events in Power Hub
    - Link to Google Calendar for editing
    Files: app/power-hub/dashboard/calendar/
    Dependencies: Google Calendar (public calendar)
    Remove if: No events/calendar needed

================================================================================
                         DATABASE SCHEMA (SUPABASE)
================================================================================

Run this SQL in Supabase SQL Editor to set up all tables:

```sql
-- Enable UUID extension
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

-- PORTAL SETTINGS (for credentials and config)
CREATE TABLE IF NOT EXISTS public.portal_settings (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  setting_key TEXT NOT NULL UNIQUE,
  setting_value JSONB NOT NULL,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.portal_settings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public access to portal_settings" ON public.portal_settings
  FOR ALL USING (true) WITH CHECK (true);

-- CUSTOM SCRIPTS (optional - for Scripts manager)
CREATE TABLE IF NOT EXISTS public.custom_scripts (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  name TEXT NOT NULL,
  location TEXT NOT NULL CHECK (location IN ('header', 'footer')),
  scope TEXT NOT NULL CHECK (scope IN ('sitewide', 'page')),
  page_path TEXT,
  content TEXT NOT NULL,
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.custom_scripts ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public access to custom_scripts" ON public.custom_scripts
  FOR ALL USING (true) WITH CHECK (true);

-- MEMBERS (optional - for Members directory)
CREATE TABLE IF NOT EXISTS public.members (
  id UUID DEFAULT uuid_generate_v4() PRIMARY KEY,
  name TEXT NOT NULL,
  category TEXT NOT NULL,
  address TEXT NOT NULL,
  phone TEXT,
  website TEXT,
  image_url TEXT,
  tier TEXT NOT NULL DEFAULT 'member' CHECK (tier IN ('sponsor', 'ambassador', 'member')),
  is_active BOOLEAN DEFAULT true,
  created_at TIMESTAMPTZ DEFAULT NOW(),
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.members ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow public access to members" ON public.members
  FOR ALL USING (true) WITH CHECK (true);
```
